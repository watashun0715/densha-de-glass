# 電車でGlass 🚃👓 設計書（Draft v0.2）

> v0.1からの変更点：GPSベース → カメラ画像処理ベースに変更。スマホカメラをグラスカメラの代替として開発開始。

---

## 概要

電車に乗るとき「何号車に乗ればいいか」をスマートグラスで教えてくれるアプリ。  
グラス（＝当面はスマホカメラ）で電光掲示板を見るだけで、目的駅への最適な乗車位置を自動案内してくれる。

---

## 解決する課題

| 課題 | スマホだと… | グラスだと… |
|------|------------|------------|
| 乗車位置がわからない | アプリを開いて検索する手間 | 掲示板を見るだけで自動通知 |
| 荷物が多いとき | 両手が塞がって見れない | 音声だけで案内 |
| 足の不自由な方 | エレベーターの場所を毎回調べる | 事前設定で自動最適化 |

---

## ユーザーストーリー

```
ユーザーが目的駅と移動スタイルを事前設定
　↓
ホームでグラス（スマホ）を電光掲示板に向ける
　↓
OCRが「山手線 新宿方面 3番線」を認識
　↓
乗車位置計算エンジンが最適号車を算出
　↓
グラスに「3号車・前扉付近 / エレベーターまで最短」と表示
```

---

## システム構成

```
[スマホカメラ] ※将来：グラスカメラに置き換え
    ↓ 画像
[Cloud Vision OCR API]
    ↓ テキスト「山手線 新宿方面 3番線」
[路線・駅情報パーサー]
    ↓ 路線名・行き先・番線
[乗車位置計算エンジン]
    ↓ 最適号車・ドア位置
[表示モジュール] ※将来：MentraOS SDK 経由でグラスに送信
    ↓
[スマホ画面 or グラス]
```

## 開発ステップ

### Step 1：OCRで掲示板を読む（Python）
```
撮影した画像 → Cloud Vision API → テキスト抽出
「山手線」「新宿方面」「3番線」をパース
```

### Step 2：乗車位置を計算する（Python）
```
路線名 + 目的駅 → Yahoo! API → 停車位置データ取得
移動スタイルと照合 → 最適号車・ドアを出力
```

### Step 3：スマホ画面に表示する
```
「3号車・前扉付近 / エレベーターまで最短」
CLIでもFlaskの簡易UIでもOK
```

### Step 4：グラス通知に切り替える（TypeScript）
```
Step 3の表示部分をMentraOS SDKに置き換えるだけ
ロジックはStep 1〜2をそのまま流用
```
